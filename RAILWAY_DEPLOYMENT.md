# Railway Deployment Guide

## Prerequisites
1. Railway account (https://railway.app)
2. Git repository with your code
3. PostgreSQL database service on Railway

## Deployment Steps

### 1. Create Railway Project
1. Go to https://railway.app
2. Click "New Project"
3. Choose "Deploy from GitHub repo" or "Empty Project"

### 2. Add PostgreSQL Database
1. In your Railway project, click "New Service"
2. Select "Database" → "PostgreSQL"
3. Railway will automatically create the database and provide connection details

### 3. Configure Environment Variables
In your Railway project settings, add these environment variables:

**Required Variables:**
- `DATABASE_URL` - (Auto-generated by Railway PostgreSQL service)
- `SECRET_KEY` - Your JWT secret key
- `PGUSER` - postgres (from Railway)
- `PGPASSWORD` - (from Railway PostgreSQL service)
- `PGDATABASE` - railway (from Railway)

**Optional Variables:**
- `UPLOAD_DIR` - uploads (for file storage)

### 4. Deploy Your Application
1. Connect your GitHub repository to Railway
2. Railway will automatically detect it's a Python app
3. It will use the `Procfile` and `railway.json` for configuration
4. The build will install dependencies from `requirements.txt`

### 5. Create Admin User (After Deployment)
Once deployed, you can create an admin user by running:
```bash
railway run python create_admin.py
```

Or use Railway's console to execute the script.

## Important Files for Railway

### Procfile
```
web: uvicorn main:app --host 0.0.0.0 --port $PORT
```

### railway.json
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "python -m uvicorn main:app --host 0.0.0.0 --port $PORT --workers 1",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

## API Endpoints
After deployment, your API will be available at:
- Main API: `https://your-app.railway.app`
- Documentation: `https://your-app.railway.app/docs`
- Health Check: `https://your-app.railway.app/health`

## Troubleshooting

### Common Issues:
1. **Database Connection**: Make sure PostgreSQL service is running and environment variables are set
2. **Port Issues**: Railway automatically sets the PORT variable
3. **Dependencies**: Ensure all packages are in `requirements.txt`

### Logs:
Check Railway deployment logs for any errors during build or runtime.

## Security Notes
- Update CORS origins for production
- Use strong SECRET_KEY
- Consider rate limiting for production use
